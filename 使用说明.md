# 环信 IM 插件 — 使用说明

## 流程概览

- **用户发消息** → 环信服务器 → **发送后回调** POST 到你的插件 → 插件校验签名、取出文本 → 调用 OpenClaw `POST /hooks/agent`（channel=easemob, to=用户/群）
- **Agent 回复** → OpenClaw 调用插件的 **sendText** → 插件用环信 **REST API** 给对应用户或群发消息

参考：[环信 IM 文档](https://doc.easemob.com/)

---

## 一、环信侧准备

1. 在 [环信控制台](https://console.easemob.com/) 创建/进入应用，记下：
   - **组织名** `org_name`、**应用名** `app_name`
   - **Client ID**、**Client Secret**（用于 REST 鉴权）
2. **发送后回调**（收用户消息）：
   - 回调管理 → 消息回调 → 发送后回调 → 添加规则
   - 回调 URL：`http://你的公网可访问地址:18791/easemob/callback`（端口可改，需与下面配置一致）
   - 填写 **Secret**，并记下（填到 OpenClaw 配置的 `callback_secret`）
   - 勾选需要的会话类型（单聊/群组）和消息类型（至少文本）

---

## 二、OpenClaw 配置

编辑 `~/.openclaw/openclaw.json`：

```json
{
  "hooks": {
    "enabled": true,
    "token": "随便设一个密钥",
    "path": "/hooks"
  },
  "plugins": {
    "load": {
      "paths": ["/你的绝对路径/openclaw-easemob"]
    }
  },
  "channels": {
    "easemob": {
      "accounts": {
        "default": {
          "enabled": true,
          "host": "https://a1.easemob.com",
          "org_name": "你的组织名",
          "app_name": "你的应用名",
          "client_id": "环信 Client ID",
          "client_secret": "环信 Client Secret",
          "callback_secret": "环信回调规则里填的 Secret",
          "callback_port": 18791,
          "from_user": "admin"
        }
      }
    }
  }
}
```

- `callback_port`：插件用来接收环信回调的端口，需与环信控制台里的回调 URL 端口一致。
- 若网关在别的机器，`host` 用环信对应集群地址即可。

---

## 三、安装并启动

```bash
cd /path/to/openclaw-easemob
npm install
npm run build
openclaw plugins install -l .
openclaw gateway restart
```

确保运行 OpenClaw 的机器上，`callback_port`（如 18791）可被环信服务器访问（公网 IP 或域名 + 端口；本地开发可用 ngrok 等暴露）。

---

## 四、验证

1. 环信控制台里确认回调规则已启用。
2. 用环信客户端或 Demo 向你的应用发一条**文本消息**。
3. 若配置正确：插件收到回调 → 转成 Webhook → Agent 回复 → 插件通过 REST 把回复发回环信，你在客户端应能看到回复。

排查可看 `openclaw logs` 里是否有 `[easemob]` 相关日志；回调校验失败会返回 403，未配 `callback_secret` 时不会校验签名。

**若出现 401**：表示插件请求 OpenClaw 的 `/hooks/agent` 时鉴权失败。请确认：
1. `openclaw.json` 里 **hooks.enabled** 为 **true**，**hooks.token** 已填写且与 Gateway 实际使用的 token 一致；
2. 修改配置后需 **重启 Gateway**（`openclaw gateway restart`）；
3. 也可用环境变量 **OPENCLAW_HOOKS_TOKEN** 传入 token（插件会优先使用）。
